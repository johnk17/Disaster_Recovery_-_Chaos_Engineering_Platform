Chaos Engineering
4 Failure Scenarios
Deliberately injecting real-world failures to prove the system recovers automatically — the same approach used by Netflix, Google, and Amazon.
🔪
Pod Kill
Randomly terminates a running container to verify Kubernetes self-healing. The killed pod is automatically replaced within seconds.
PodChaos mode: one recovery: 26s
🔥
CPU Stress
Spikes CPU utilization to 80% on multiple pods to test resource limits, autoscaling behavior, and application degradation under load.
StressChaos load: 80% duration: 2m
🌐
Network Delay
Injects 200ms ± 50ms latency into outgoing traffic to simulate degraded network conditions and test timeout/retry configurations.
NetworkChaos latency: 200ms duration: 3m
💀
DNS Failure
Completely breaks DNS resolution to simulate a catastrophic infrastructure failure. Tests application resilience when dependent services become unreachable.
DNSChaos pattern: * duration: 1m
